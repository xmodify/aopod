################## Almalinux 9.4 (AOPOD Setup Guide) ###################

คู่มือการติดตั้งระบบ AOPOD (Laravel 12 / PHP 8.2) บน AlmaLinux 9 
โดยอ้างอิงโครงสร้างสิทธิ์การใช้งานแบบเดียวกับโครงการที่ผ่านการรับรองแล้ว (Apache Owner Model)

--------------------------------------------------------------------------------
📌 1. เตรียมระบบพื้นฐาน (ถ้ายังไม่ได้ลง)
--------------------------------------------------------------------------------
sudo dnf update -y
sudo dnf install -y git unzip

--------------------------------------------------------------------------------
📌 2. ติดตั้ง Apache + PHP (Laravel 12 แนะนำ PHP >= 8.2)
--------------------------------------------------------------------------------
sudo dnf install -y epel-release
sudo dnf install -y https://rpms.remirepo.net/enterprise/remi-release-9.rpm

# เปิดใช้งานโมดูล PHP 8.2 จาก REMI Repository (ต้องการ PHP >= 8.2 สำหรับ Laravel 12)
sudo dnf module enable -y php:remi-8.2
sudo dnf install -y php php-cli php-common php-mbstring php-xml php-bcmath php-curl php-zip php-mysqlnd php-pdo php-tokenizer php-opcache unzip git curl

--------------------------------------------------------------------------------
📌 3. ติดตั้ง Composer
--------------------------------------------------------------------------------
cd /tmp
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php composer-setup.php
sudo mv composer.phar /usr/local/bin/composer
composer --version

--------------------------------------------------------------------------------
📌 4. Clone Project ลง /var/www/html/aopod
--------------------------------------------------------------------------------
cd /var/www/html
sudo git clone https://github.com/xmodify/aopod.git aopod

--------------------------------------------------------------------------------
📌 5. ตั้ง permission ให้ Apache ใช้งานได้
--------------------------------------------------------------------------------
sudo chown -R apache:apache /var/www/html/aopod
sudo chmod -R 775 /var/www/html/aopod
sudo chmod -R 775 /var/www/html/aopod/storage
sudo chmod -R 775 /var/www/html/aopod/bootstrap/cache

--------------------------------------------------------------------------------
📌 6. ติดตั้ง Laravel dependencies
--------------------------------------------------------------------------------
cd /var/www/html/aopod
sudo -u apache /usr/local/bin/composer install

--------------------------------------------------------------------------------
📌 7. ตั้งค่าไฟล์ .env
--------------------------------------------------------------------------------
sudo -u apache cp .env.example .env
sudo -u apache php artisan key:generate
sudo nano .env

--------------------------------------------------------------------------------
📌 8. git pull ผ่าน User (คำสั่งอัปเดตและเคลียร์แคชระบบ)
--------------------------------------------------------------------------------
sudo -u apache git -C /var/www/html/aopod reset --hard
sudo -u apache git -C /var/www/html/aopod pull origin main
sudo -u apache /usr/local/bin/composer install --no-dev --optimize-autoloader
sudo -u apache rm -f /var/www/html/aopod/bootstrap/cache/*.php
sudo -u apache php artisan optimize:clear
	
--------------------------------------------------------------------------------
📌 9. ตั้งค่า Apache Web Server
--------------------------------------------------------------------------------
sudo dnf install -y httpd
sudo systemctl enable httpd
sudo systemctl start httpd

--------------------------------------------------------------------------------
📌 10. Node.js Setup (สำหรับสร้างไฟล์ Frontend assets):
--------------------------------------------------------------------------------
sudo dnf module reset nodejs -y
sudo dnf module enable nodejs:20 -y
sudo dnf install -y nodejs
cd /var/www/html/aopod
sudo -u apache npm install
sudo -u apache npm run build

###########################################################
# 11. การตั้งค่าแบบ Option B (แนะนำ - เชื่อมพอร์ตเสมือนผ่าน Proxy)
# * ป้องกันพอร์ตชนกัน: rims ใช้พอร์ต 8883, โครงการ aopod จึงใช้พอร์ต 8884 *
###########################################################

1. แก้ไขไฟล์ VirtualHost ของพอร์ต 8884 (เช่น /etc/httpd/conf.d/aopod.conf):
   Listen 8884
   <VirtualHost *:8884>
       ServerName 192.168.0.4
       Alias /aopod "/var/www/html/aopod/public"
       DocumentRoot "/var/www/html/aopod/public"
       <Directory "/var/www/html/aopod/public">
           Options FollowSymLinks
           AllowOverride All
           Require all granted
           FallbackResource /aopod/index.php
       </Directory>
   </VirtualHost>

2. แก้ไขไฟล์ Proxy (พอร์ต 80) ในไฟล์ config หลักของ Apache เพื่อชี้ลิ้งค์ภายนอก:
   <VirtualHost *:80>
       ServerName 192.168.0.4

       # แก้ไขปัญหา 404 ด้วย Location Block สำหรับ aopod
       <Location /aopod>
           ProxyPass http://127.0.0.1:8884/aopod
           ProxyPassReverse http://127.0.0.1:8884/aopod
       </Location>
       
       ProxyPassReverseCookiePath /aopod /aopod
   </VirtualHost>

3. คำสั่งสำคัญ เพื่อไม่ให้ติด 404 (SELinux และ Firewall):
   sudo setsebool -P httpd_can_network_connect 1
   sudo semanage port -a -t http_port_t -p tcp 8884
   sudo firewall-cmd --permanent --add-port=8884/tcp
   sudo firewall-cmd --reload

4. ตั้งค่าในไฟล์ .env บนเซิร์ฟเวอร์จริง:
   APP_URL=http://192.168.0.4/aopod

5. รีสตาร์ท Apache:
   sudo systemctl restart httpd

--------------------------------------------------------------------------------
📂 ไฟล์แนบ /public/.htaccess มาตรฐานความปลอดภัยสูง:
--------------------------------------------------------------------------------
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [L]
</IfModule>
